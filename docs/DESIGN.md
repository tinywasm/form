# Design & Architecture

## Philosophy
- **TinyGo first**: Zero `reflect`, flat slices over maps, minimal allocations.
- **Explicit over implicit**: Input types assigned via `fmt.Widget` in schema (generated by `ormc`). No magic name matching.
- **Interface-driven**: Decoupled from concrete structs via `fmt.Fielder`.

## Core Layers

### 1. Widget Resolution (`form.go`)
`New()` iterates `data.Schema()`. Each `fmt.Field` carries a `Widget` (set by `ormc`):
- `field.Widget == nil` → field skipped (no UI binding).
- `field.Widget.Clone(formID, fieldName).(input.Input)` → positioned input ready for rendering.
- No registry lookup. No name-based heuristics. The schema is the source of truth.

`RegisterInput()` and `findInputByType()` remain available for explicit runtime lookup by projects that need it, but are not called automatically.

### 2. State & Binding (`form.go`, `sync.go`)
- **Fielder-based**: `New()` uses `data.Schema()` and `data.Pointers()`.
- **Pointer-based sync**: `SyncValues(data)` uses `data.Pointers()` to write back values without reflection.
- Supports `fmt.FieldText`, `fmt.FieldInt`, `fmt.FieldFloat`, `fmt.FieldBool`.

### 3. Validation (`validate.go`, `validate_struct.go`)
- `Form.Validate()` iterates `f.Inputs`, calls `inp.Validate(GetSelectedValue())`.
- Skips fields with `SkipValidation` flag.
- Returns first error encountered.
- `ValidateData(action, data)` provides server-side or isomorphic validation via `crudp.DataValidator`.

### 4. Rendering (`render.go`)
`RenderHTML()` builds: `<form id="..."> [method action if SSR] inputs [submit if SSR] </form>`

### 5. WASM Interactivity (`mount.go`)
`OnMount()` attaches delegated listeners:
- `input`/`change` → Live sync and validation.
- `submit` → `SyncValues(f.data)` → `Validate()` → `OnSubmit(f.data)`.

One listener per form (not per input) = fewer closures = smaller WASM binary.
