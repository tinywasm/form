# tinywasm/form
<img src="docs/img/badges.svg">

Fast form generation from Go structs for TinyGo + WASM. Uses `model.Fielder` interface for zero-reflection data binding. Input types are assigned via `model.Field.Type` in the schema (generated by `ormc`).

## Rules (Non-Negotiable)

- **No stdlib imports** in library code. Never `errors`, `strconv`, `strings`, `reflect`.
- **Use `github.com/tinywasm/fmt`**: `fmt.Err("Noun", "Adjective")` for errors, `fmt.Convert(s).Int()` instead of `strconv.Atoi`, `fmt.Contains(s, sub)` instead of `strings.Contains`.
- **No maps in inputs**: Use slices. Maps increase WASM binary size.
- **No `reflect` at runtime.**

## Quick Start

```go
// User struct implements model.Fielder (generated by ormc).
// Each field in Schema() has a Kind assigned (e.g. input.Email()).
f, err := form.New("parent-id", &User{Name: "John"})
html := f.String()    // render to HTML string
```

## How Field Matching Works

`form.New()` iterates `data.Schema()`. For each field:

1. A field is a form field iff its `model.Kind` (the `Type` field) also implements `input.Input`.
2. Fields with base kinds (`model.Text()`, etc.) are not rendered in the form but still validate.
3. `field.Type.(input.Input).Clone(formID, fieldName)` creates a positioned input instance.
4. Constraint-based defaults are applied (e.g. `NotNull` â†’ `SetRequired(true)`).
5. Current value from `data.Pointers()` is bound to the input.

There is **no registry-based name matching**. Each field's kind is set explicitly in the schema by `ormc`.

## Input Interface

```go
// input.Input â€” defined in input/interface.go
type Input interface {
    model.Kind  // Name(), Storage(), Validate(value string) error
    Clone(parentID, name string) Input
    FieldName() string
    SetRequired(bool)
    AddAttribute(key, value string)
    // Metadata getters used by form.RenderInput:
    GetID() string
    SetID(string)
    GetValues() []string
    GetOptions() []fmt.KeyValue
    GetPlaceholder() string
    GetTitle() string
    IsRequired() bool
    IsDisabled() bool
    IsReadonly() bool
    GetAttributes() []fmt.KeyValue
    ErrorID() string
    HTMLName() string
    HandlerName() string
}
```

`input.Input` does **not** embed `dom.Component`. HTML rendering is done by `form.RenderInput(input.Input)` in the `form` package, keeping `input` free of `dom`/`html` imports (edge-safe).

Each concrete type (Email, Text, etc.) provides:
- `Xxx() Input` â€” template instance for use in schema (no position).
- `Clone(parentID, name string) Input` â€” creates a positioned copy ready for rendering.
- `Name() string` â€” semantic input type name (e.g. `"email"`).
- `Storage() model.FieldType` - deterministic storage mapping.
- `Validate(value string) error` â€” validates input value via `Permitted` rules.

## Public API

### `form.New(parentID string, data model.Fielder) (*Form, error)`

Creates a `Form` from a `Fielder`. Uses `data.Schema()` to resolve inputs via `field.Type`. Form `id` = `parentID + "." + resolveStructName(data)`.

### `form.SetGlobalClass(classes ...string)`

Sets CSS classes applied to all new forms.

### Form Methods

| Method | Description |
|--------|-------------|
| `GetID() string` | Returns form's HTML id |
| `SetSSR(bool) *Form` | Enables SSR mode (adds method, action, submit button) |
| `String() string` | Generates form HTML |
| `Validate() error` | Validates all inputs, returns first error |
| `SyncValues(model.Fielder) error` | Copies input values back into the provided data |
| `ValidateData(byte, model.Fielder) error` | Server-side validation (crudp.DataValidator) |
| `OnSubmit(func(model.Fielder, func(error))) *Form` | Sets WASM submit callback |
| `Render() *dom.Element` | **WASM only** â€” returns reactive DOM tree |
| `Input(fieldName string) input.Input` | Returns input for a field name |
| `SetOptions(fieldName, ...fmt.KeyValue) *Form` | Sets options for select/radio/datalist |
| `SetValues(fieldName, ...string) *Form` | Sets value programmatically |

### Reactive Rendering

`Form` implements `dom.ViewRenderer`, providing a `Render() *dom.Element` method that returns a reactive DOM tree with signal-bound inputs. This ensures IME-safe live validation and surgical DOM updates without full re-renders.

### Namer Interface

Fielder types can optionally implement `Namer` to customize the form name:
```go
type Namer interface {
    FormName() string
}
```
If not implemented, defaults to `"form"`.

## WASM Event Flow

```
dom.Mount("root", f)
  -> f.Render() returns a reactive dom.Element tree
  -> Inputs are bound to signals via Bind()
  -> Live validation updates error signals on "input"
  -> Submit button state bound to submitting signal
```

Reactive binding ensures IME/cursor safety during live validation.

## Struct Tags

The unified `input:` tag is parsed by `ormc` at generation time to populate the `model.Field` schema:

| Feature | Format | Example |
|---------|--------|---------|
| **Widget** | `input:"type"` | `input:"email"`, `input:"textarea"`, `input:"-"` |
| **Validation** | `input:"rule"` | `input:"required"`, `input:"min=5"`, `input:"max=100"` |
| **Metadata** | `input:"key=val"` | `input:"title=\"Label\""`, `input:"placeholder=\"Hint\""` |
| **Options** | `input:"options=..."` | `input:"options=\"1:Yes,0:No\""` |

Multiple rules are comma-separated: `` `input:"email,required,max=100"` ``. See [docs/TAGS.md](docs/TAGS.md) for full reference.

Runtime overrides: `f.Input("Field").SetPlaceholder()`, `f.SetOptions("Field", ...)`.

## Validation Engine (`fmt.Permitted`)

```go
type Permitted struct {
	Letters    bool     // a-z, A-Z (and Ă±/Ă‘)
	Tilde      bool     // Ăˇ, Ă©, Ă­, Ăł, Ăş
	Numbers    bool     // 0-9
	Spaces     bool     // ' '
	BreakLine  bool     // '\n'
	Tab        bool     // '\t'
	Extra      []rune   // additional allowed characters
	NotAllowed []string // disallowed substrings
	Minimum    int      // minimum length
	Maximum    int      // maximum length
}
```
